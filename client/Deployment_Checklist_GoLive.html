1. Pre-Deployment (Client)

Complete before Phase 1 production deployment:

Infrastructure

  • Linux server provisioned (PHP 8.2+, MySQL 8.x, Nginx/Apache)
  • HTTPS certificate installed and auto-renewal configured
  • Domain DNS pointing to server (crm.alsultania.ae or agreed URL)
  • Staging environment available for UAT
  • SSH/deploy access granted to AnyMatch
  • Firewall allows outbound HTTPS (WhatsApp, SMS, SMTP APIs)
  • Queue worker process configured (Supervisor or systemd)
  • Cron entry for Laravel scheduler (* * * * * php artisan schedule:run)
  • Private storage directory outside web root (NFR-07)
  • Daily database backup job scheduled (NFR-10)

Third-party services

  • WhatsApp Business API verified and webhook URL registered
  • SMS gateway account with credits for OTP
  • SMTP credentials for transactional email
  • All credentials securely transferred (not via plain email)

See full detail: SRS Appendix D — Third-Party Services (companion PDF)

2. Application Deployment

2.1 Initial release

# On server (example)
git pull origin main
composer install --no-dev --optimize-autoloader
npm ci && npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan storage:link
php artisan queue:restart

2.2 Environment file

  • APP_ENV=production
  • APP_DEBUG=false
  • APP_URL set to production HTTPS URL
  • Database credentials configured
  • APP_KEY generated and secured
  • Mail, SMS, WhatsApp vars set (or configured via Channel Settings post-deploy)

2.3 Post-deploy verification

  • Login page loads over HTTPS
  • Admin can log in
  • Login page loads; guest inquiry tab OTP sends (test number)
  • WhatsApp webhook receives test message
  • Queue processes jobs (php artisan queue:work running)
  • Scheduler runs (check storage/logs/)

3. Phase-Specific Go-Live Gates

Phase Gate Required integrations
Phase 1 CRM + leads + public inquiry live WhatsApp, SMS, SMTP
Phase 2 Finance modules enabled SMTP for invoices
Phase 3 Automation + multi-branch All channels + 2+ branches seeded

Do not enable Phase 2/3 routes until prior phase UAT signed off (SRS §9).

4. Security Hardening

  • APP_DEBUG=false in production
  • Directory listing disabled
  • .env not web-accessible
  • File upload size limits configured
  • Rate limiting active on guest inquiry OTP routes (NFR-12)
  • Session lifetime configured (NFR-08)
  • Admin default password changed after first login

5. Monitoring & Operations

Item Recommendation
Application logs storage/logs/laravel.log + log rotation
Queue failures Monitor failed_jobs table
Uptime External ping on /login
Backups Verify restore procedure before go-live
SSL expiry Calendar reminder 30 days before renewal

6. Rollback Plan

  1. Keep previous release tag available
  2. php artisan migrate:rollback only if migration is reversible
  3. Restore database from pre-deploy backup if data migration failed
  4. Document rollback decision in project channel

7. Handover Deliverables

Per phase completion:

  • Production URL and admin credentials (secure channel)
  • .env template (without secrets)
  • Database backup export
  • User guide or training session (optional per contract)
  • 14-day validation period start date recorded
  • 30-day warranty period documented (NFR-13)
