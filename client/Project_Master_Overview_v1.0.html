2. What This Project Is

A full-stack travel agency CRM that unifies:

  • Lead capture, assignment, and pipeline management — including guest inquiry on login page (phone OTP → required fields → auto-assign to next available agent)
  • Customer profiles and booking workflows
  • Financial accounting, invoicing, and expense controls
  • Staff performance and HR metrics
  • WhatsApp automation and multi-channel notifications
  • Reporting, analytics, and multi-branch scalability

Phase 0 (current) delivers the approved blueprint: screen flows, data model, component library, and requirements specification. Development (Phases 1–3) begins only after client approval of the Figma blueprint.

Guest inquiry flow (Phase 1)

Guests submit leads on the login page (Submit inquiry tab) without CRM login. Staff use the same page with email and password only. Full spec: requirements/SRS.md Appendix C §1a.

Step Prototype What happens
1 login.html (01 — guest tab) Guest enters mobile number; system sends 6-digit SMS OTP
2 login.html (01 — guest tab) Guest verifies OTP; verified phone stored in session
3 inquiry-create.html (05c) Guest completes required fields (name, service type, type-specific fields, dates & travelers)
4 System (AUTO-02) Lead saved with lead_via = Website; auto-assigned to next available active agent
5 inquiry-success.html (05d) Guest sees confirmation with reference ID (agent assignment is internal)

Staff Create Lead (leads-create.html) uses the same field model but requires CRM login and manual Lead Handling (Takeover / Others / Transfer).

7. Delivery Phases & Modules

Phase 1 — Core CRM (Screens 01–12)

Authentication, RBAC, dashboard, advanced lead management (staff create + guest inquiry on login page with OTP and auto-assignment), pipeline, WhatsApp inbox, customers, bookings.

Key specs: SRS §4.1 (FR-18*, AUTO-02), SRS Appendix C §1a (public), Appendix C §2 (staff)

Phase 2 — Financials & HR (Screens 13–23)

Accounting dashboard, revenue, invoices, payments, financial reports, expenses, approvals, staff performance, attendance, cost centres.

Key specs: SRS §4.2

Phase 3 — Automation & Scale (Screens 24–33)

Reports hub, custom report builder, notifications, communication templates, channel settings, automation rules, workflow triggers, multi-branch, consolidated view, system settings.

Key specs: SRS §4.3, §5 (automation)

User roles

Role Primary modules
Administrator All modules, settings, automation, branches
Sales Manager Leads, team, reports, approvals
Travel Agent Leads, customers, bookings, WhatsApp
Finance User Accounting, invoices, expenses (Phase 2)
HR/Manager Performance, attendance (Phase 2)

Full RBAC matrix: SRS §3.1

11. Change Management

After Phase 0 blueprint approval, the accepted SRS and Figma blueprint are the locked baseline for development.

  • Post-approval changes to screens, fields, flows, or scope require written agreement between Client and Vendor
  • Commercial terms and pricing remain governed by the accepted proposal
  • Client dependencies (server, WhatsApp, SMS, SMTP) must be ready per the timelines in the SRS

For questions about scope or acceptance criteria, refer to the SRS §9 (Phase-wise Delivery & Acceptance).
